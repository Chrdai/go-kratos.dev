Kratos

Get Started

(en) API规范化

(en) API 协议使用 Protobuf 定义接口，实现 HTTP/gRPC 方式进行通信， 并且 Errors 通过 Enum 作为错误码，以实现错误判定。

(en) 组件插件化

(en) Component 统一遵循 OpenTelemetry 规范进行实现， 通过 Plugins 设计理念，实现插件化方式提供扩展能力。

(en) 项目工程化

(en) 通过 Git 方式管理 Service 项目模板，并可通过 Kratos 一键初始化脚手架。
